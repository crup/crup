I build and lead teams around the engineering systems that make products scale: reliable APIs, clear service boundaries, cloud-native operations, frontend platforms teams can extend, and tooling that makes quality the default.

My work usually sits between service design, distributed systems, runtime decisions, React / Next.js platform architecture, web performance, design systems, and developer experience. I like staying close to the code while leading teams, shaping architecture, coaching engineers, and helping product, design, backend, data, and engineering teams land the right tradeoffs.

### 🧭 What I Care About

- ⚙️ Backend systems with clear service boundaries, reliable APIs, observable behavior, and room to scale without constant rewrites.
- 🧱 Product platforms that let teams move independently while still sharing the right standards, libraries, and delivery practices.
- ⚡ Web performance as an everyday engineering habit: fast pages, stable layouts, Core Web Vitals, SEO-friendly rendering, and fewer regressions after release.
- 🔍 Quality loops that fit real delivery: tests where they matter, useful monitoring, practical reviews, and release checks that catch the right problems.
- 🧠 Engineering leadership that stays close to technical decisions, system design, mentorship, planning, and cross-functional tradeoffs.
